# PRD: EduPlan AI

## 1. 문서 개요

- 제품명: `EduPlan AI (에듀플랜 AI)`
- 제품 정의: 2022 개정 교육과정과 개념 기반 탐구학습(Transfer 중심)을 바탕으로 교사의 수업 설계 전문성을 보조하는 AI 기반 커리큘럼 디자이너
- 문서 목적: 개발팀/디자이너/기획자가 동일한 기준으로 즉시 구현 착수할 수 있는 실행 중심 요구사항 정리

## 2. 제품 비전 및 목표

- 단순 수업 도우미를 넘어 교사의 판단과 설계를 강화하는 `Curriculum Designer` 제공
- 성취기준 기반으로 단원-개념-탐구질문-평가-차시를 일관된 맥락으로 설계
- 단계별 확정 결과를 누적해 AI 응답의 일관성과 재현성을 확보

### 성공 기준

- 교사가 성취기준 선택부터 차시 설계/문서 출력까지 단일 흐름에서 완료 가능
- 단계 간 맥락 단절 없이 결과물 품질 유지
- 교사 수작업 대비 설계 시간 단축 및 품질 균질화

## 3. 사용자 및 핵심 시나리오

### 1차 사용자

- 초등학교 교사
- 학교 단위 연구부/교과부장(파일럿 단계)

### 핵심 사용 시나리오

1. 교사가 성취기준을 탐색/선택한다.
2. AI와 협업해 5단계 설계를 진행한다.
3. 각 단계 결과를 확정하고 수정한다.
4. 평가/차시 설계를 생성하고 문서로 출력한다.
5. 추후 재진입 시 기존 확정 맥락을 불러와 계속 편집한다.

## 4. 정보 구조 및 시스템 아키텍처

플랫폼의 핵심은 `이전 단계 결과가 다음 단계 입력 컨텍스트로 강제 주입`되는 연쇄 구조다.

### Data Layer

- 2022 개정 교육과정 성취기준 저장소(JSON/관계형 DB/Vector DB)
- 교사 업로드 커스텀 파일(PDF/Excel 텍스트 추출 결과)

### Logic Layer

- 단계별 프롬프트 체이닝(Prompt Chaining)
- RAG 기반 성취기준 검색/주입
- 단계 확정 상태 기반 Lock & Load 로직

### Storage Layer

- 단계별 확정 데이터(`step1~step5`) 저장
- 대화 로그, 확정 카드, 버전 이력 저장
- 사용자/학교 단위 접근 제어 및 감사 로그 저장

## 5. 기능 요구사항 (Functional Requirements)

### FR-1. 성취기준 연동 모듈

- 학년군/교과/영역 필터 기반 성취기준 브라우저 제공
- 성취기준 카드를 코드/본문/교과정보와 함께 표시
- 키워드 검색 시 다교과 통합 추천(RAG)
- 예: `"환경"` 검색 시 과학/사회/도덕 관련 성취기준 동시 제시

### FR-2. 커스텀 교육과정 업로드

- PDF/Excel 업로드 후 텍스트 추출
- 추출 결과를 성취기준 후보로 가공 및 선택 가능하게 제공
- 학교장 개설 과목/학교 자체 교육과정 반영 가능

### FR-3. 3-Pane 인터페이스

#### 좌측: Step Navigation

- 단계 상태 표시: `진행 중 / 완료(잠금) / 미진행`
- 단계 클릭 시 해당 단계 대화 기록 및 산출물로 즉시 이동
- 전체 진행률(완성도) 시각화

#### 중앙: AI Design Lab

- 단계별 Guide Chips 제공
- 예: `핵심 개념 제안해줘`, `탐구 질문 다듬어줘`
- 실시간 생성 스트리밍 표시
- 응답 중단/재요청/수정 요청 지원

#### 우측: Outcome Archive

- 확정 트리거 시 요약 카드 자동 생성
- 인라인 편집(직접 타이핑) 가능
- 편집 결과는 다음 단계 컨텍스트에 자동 반영

### FR-4. Lock & Load 맥락 엔진

- 단계 완료 시 상태 JSON 업데이트
- 예: `{ step1: ..., step2: ..., step3: ..., step4: ..., step5: ... }`
- 이후 단계 프롬프트 상단에 확정 내용 강제 주입
- 예: `[기확정 내용: {step1}, {step2}]`
- 확정 단계의 무분별한 변형 방지를 위한 잠금/해제 정책 제공

### FR-5. 단계별 AI 워크플로우

#### 1단계. 단원 설정

- 입력: 선택 성취기준
- 내부 로직: 명사(내용 요소)/동사(수행) 추출
- 출력: 성취기준 코드, 단원명, 학습 목표

#### 2단계. 개념 추출

- 입력: 1단계 확정 결과
- 내부 로직: 사실 나열이 아닌 전이 가능한 핵심 개념 도출
- 출력: 핵심 개념(예: 시스템, 상호작용)

#### 3단계. 탐구 질문

- 입력: 1~2단계 확정 결과
- 내부 로직: 개념 이해를 자극하는 질문 생성
- 출력: 본질적 질문 1개, 단원 질문 3개

#### 4단계. 평가 설계

- 입력: 1~3단계 확정 결과
- 내부 로직: GRASPS 기반 수행 과제 및 루브릭 구성
- 출력: 수행 시나리오, 평가 기준표(루브릭)

#### 5단계. 차시 설계

- 입력: 1~4단계 확정 결과
- 내부 로직: WHERETO 전략 기반 활동 설계
- 출력: 1~8차시 상세 계획, AI 도구 활용 제안

### FR-6. 산출물 생성/내보내기

- 수업 설계 산출물을 학교 현장 양식 중심으로 구성
- 출력 포맷: 우선 PDF/DOCX, 이후 HWP 확장
- HWP는 표 기반 단원 설계안 레이아웃 유지

### FR-7. 저장/이력/재진입

- 프로젝트 단위 자동 저장(Auto-save)
- 확정 카드/대화/단계 결과 버전 관리
- 재접속 시 마지막 작업 상태 복구

## 6. 비기능 요구사항 (Non-Functional Requirements)

### 성능

- 주요 상호작용은 교사가 지연을 체감하지 않도록 최적화
- 장문 생성은 스트리밍 기반으로 초기 응답 체감 속도 확보

### 일관성/품질

- LLM 파라미터는 논리성과 재현성 중심
- 권장 Temperature 범위: `0.3~0.5`
- 단계별 출력 스키마 검증(질문 수, 루브릭 구조 등)

### 보안/개인정보

- 민감 데이터 암호화 저장
- AI 학습 데이터로 재사용되지 않음을 명시
- 사용자/학교 단위 데이터 접근 제어
- 감사 로그(Audit Log) 기반 추적성 확보

## 7. UX/UI 가이드

### 시각 스타일

- 교사 피로도를 낮추는 Mint/Slate Blue 계열 중심 팔레트
- 정보 밀도는 높되 시선 흐름이 분명한 3-Pane 레이아웃

### 상호작용 피드백

- 단계 완료 시 프로그레스 바 갱신
- 수업 설계 완성도 점수 시각화
- 확정/저장/생성/오류 상태를 명확히 노출

### 편집 경험

- AI 출력은 항상 수정 가능한 초안으로 제시
- 인라인 편집 결과가 즉시 컨텍스트에 반영됨을 안내

## 8. AI 오케스트레이션 및 가드레일

- Provider 추상화(OpenAI/Anthropic 등)로 확장성 확보
- 단계별 시스템 프롬프트 템플릿 분리 관리
- 출력 JSON 스키마 강제 및 파싱 실패 시 폴백 처리
- 이전 단계 확정값 누락/충돌 시 명시적 경고 및 재생성 유도

## 9. 운영 및 측정 지표

### 제품 지표

- 설계 완료율(시작 대비 최종 산출물 생성 비율)
- 단계별 이탈률(특히 3→4, 4→5 구간)
- 산출물 재편집률(AI 초안의 현장 적합성 지표)

### 품질 지표

- 루브릭 구조 준수율
- 질문 품질 기준 충족률(본질적 질문/단원 질문 형식)
- 교사 만족도(유용성, 신뢰성, 시간 절감 체감)

## 10. 범위 정의

### 포함 범위

- 성취기준 기반 5단계 설계 흐름
- 3-Pane 상호작용 및 확정 카드 아카이브
- Lock & Load 기반 컨텍스트 연계
- 평가/차시 산출 및 문서 출력

### 확장 범위

- 고급 RAG 추천 정밀도 향상
- 학교/교육청 관리자 기능
- HWP 고급 템플릿 및 외부 LMS 연동

## 11. 수용 기준 (Acceptance Criteria)

1. 교사는 성취기준을 선택하고 5단계를 순차 진행할 수 있어야 한다.
2. 각 단계 확정 시 결과가 우측 아카이브 카드로 생성되어야 한다.
3. 다음 단계 AI 호출 시 이전 확정 단계의 컨텍스트가 프롬프트에 주입되어야 한다.
4. 인라인 수정 내용은 저장되고 후속 단계 결과에 반영되어야 한다.
5. 최종적으로 평가/차시 결과물을 문서로 다운로드할 수 있어야 한다.
6. 오류 상황(생성 실패/파싱 실패)에서 사용자 안내와 재시도 경로가 제공되어야 한다.

## 12. 구현 우선순위

### P0 (필수)

- 3-Pane UI
- 성취기준 검색/선택
- 5단계 체인 및 Lock & Load
- 확정 카드/저장/재진입
- 기본 문서 출력

### P1 (고도화)

- 실시간 스트리밍 중단/재요청 UX
- 커스텀 파일 업로드
- 고급 RAG 추천
- HWP 템플릿 출력
